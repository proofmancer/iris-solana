Iris. Verifiable provenance
for AI agents on Solana.

Every time an AI agent takes an action through MCP, Iris writes a signed on-chain receipt to a content-addressable PDA. Any Solana program can derive the PDA from the payload hash and verify the action atomically. No indexer required.

How it works

Three primitives, one receipt.

01

Agent acts via MCP

The Iris MCP server intercepts the tool call, hashes the canonical payload, and asks the relayer to sign.

02

Relayer issues receipt

The relayer calls issue_receipt. The chain stores (action, authorizing_key, payload_hash, timestamp) at PDA [b"receipt", payload_hash].

03

Anyone verifies

Downstream programs derive the same PDA and read the account. No CPI required for the cheap path; an on-chain verify_receipt ix exists for CPI use too.

Integrate

Drop it into your Solana program.

Receipts are content-addressable. Any downstream program can verify by deriving the same PDA. 

// Cheap verification path: derive the PDA, read the account, check fields.
use anchor_lang::prelude::*;
use iris_receipts::state::{Receipt, RECEIPT_SEED};

#[derive(Accounts)]
#[instruction(payload_hash: [u8; 32])]
pub struct ExecuteWithReceipt<'info> {
    #[account(
        seeds = [RECEIPT_SEED, payload_hash.as_ref()],
        seeds::program = iris_receipts::ID,
        bump,
    )]
    pub receipt: Account<'info, Receipt>,
}

pub fn handler(ctx: Context<ExecuteWithReceipt>, payload_hash: [u8; 32]) -> Result<()> {
    let r = &ctx.accounts.receipt;
    require!(r.payload_hash == payload_hash, ErrorCode::Mismatch);
    // r.action, r.authorizing_key, r.timestamp are all on-chain truth.
    Ok(())
}
